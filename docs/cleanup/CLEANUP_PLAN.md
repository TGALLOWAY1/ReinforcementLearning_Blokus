## B. Proposed Target Structure

```
blokus_rl/
├── docs/                    # Single documentation root
│   ├── README.md             # Index linking to main docs
│   ├── architecture/        # System design, audits, optimization reports
│   ├── analytics/           # Analytics-specific docs (if any)
│   ├── training/             # Training docs (existing)
│   ├── api/                  # API/deployment docs (existing as DEPLOYMENT_*.md)
│   ├── metrics/              # Metrics spec (existing docs/metrics/)
│   └── cleanup/              # This plan + verification
├── config/                   # Agent hyperparameter configs (config/agents/*.yaml)
├── configs/                  # Training/league run configs (*.yaml)
├── scripts/                  # One scripts root
├── analytics/
├── engine/
├── webapi/
├── frontend/
├── agents/
├── league/
├── rl/
├── envs/
├── mcts/
├── schemas/
├── tests/
├── training/
├── benchmarks/
├── api-runtime/
├── engine-service/
└── (logs, checkpoints, runs - generated, consider .gitignore)
```

**Exceptions:**
- `config/` and `configs/` both kept (different purposes)
- `PR_DESCRIPTION.md` stays at root (PR-specific)
- Generated dirs: `logs/`, `checkpoints/`, `runs/`, `benchmarks/results/` — add to .gitignore if not already
